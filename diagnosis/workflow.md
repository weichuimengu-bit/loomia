## 🤖 3. Claude Code でレポート生成（15 分）

### 手順

1. **リポジトリのルートで Claude Code を起動**

   ```bash
   cd ~/path/to/loomia
   claude
   ```

2. **新ファイルを作成する**

   作業中は `diagnosis/cases/` フォルダに保存（リポジトリには commit しない）:

   ```bash
   mkdir -p diagnosis/cases
   ```

3. **以下のプロンプトを Claude Code に送る**

   ```
   diagnosis/DIAGNOSIS_PROMPT.md を読み込み、以下のクライアント情報で
   AI 活用度診断レポートを生成してください。
   出力は diagnosis/cases/LMA-{YYYYMM}-{NNN}.md に保存してください。

   ---
   client:
     業種: "{ヒアリング結果}"
     従業員数: "{ヒアリング結果 or 不明}"
     年商規模: "{ヒアリング結果 or 不明}"
   現状の悩み:
     - "{悩み1}"
     - "{悩み2}"
     - "{悩み3}"
   予算感:
     初期費用: "{ヒアリング結果 or 中央値仮置き}"
     月額: "{ヒアリング結果 or 中央値仮置き}"
   受信日: "{YYYY-MM-DD}"
   Case ID: LMA-{YYYYMM}-{NNN}
   ```

4. **Claude が生成した Markdown を確認**
   - 通常 1 回の生成で 90% の品質に到達
   - TEMPLATE.md の構造に従っているか ざっと確認

## 👀 4. レビュー・微調整（10 分）

### 必須チェック（5 分）

- [ ] **事実誤認がないか**: ツール名のスペル、価格、業界固有の用語
- [ ] **価格の整合性**: Loomia のサービス価格帯（Layer 03 初期30-150万円 /
      月額5-20万円）と矛盾していないか
- [ ] **トーン**: 誇大広告になっていないか、クライアントを見下していないか
- [ ] **5 ページの分量バランス**: どこか極端に薄い/厚いページはないか
- [ ] **CTA の着地**: 最終ページが Loomia のサービスメニューに
      自然に接続されているか

### よくある修正ポイント

| 症状 | 対処 |
|------|-----|
| 抽象論が多い | 「具体ツール名と数値で書き直して」と Claude に再指示 |
| 「織る/光」比喩が多すぎる | 「比喩を最大 2 箇所に減らして」 |
| 価格がちぐはぐ | 「Layer 03 は初期30万円〜、月5万円〜で統一して」 |
| クライアント名が抜けている | メタブロックと 1 ページ目で明示する |

### 偉吹の固有色を足す（5 分、任意）

- そのクライアント向けに **1-2 行の手書きコメント** を 1 ページ目冒頭に差し込む
  と差別化になる（例:「個人サロン向けの診断は 3 件目ですが、毎回思うのは...」）

## 📄 5. PDF 化（5 分）

### 推奨: Claude 公式 pdf skill を使う（最速）

Claude Code の skill 一覧に `pdf` がある場合:

```
diagnosis/cases/LMA-{YYYYMM}-{NNN}.md を PDF に変換してください。
- A4 縦
- マージン: 上下 20mm / 左右 18mm
- 各 H2 見出し（## ページ X）の前でページ区切り
- アクセントカラー: #7dd3fc
- フォント: 日本語は Noto Sans JP、英字は Inter
- 出力先: diagnosis/cases/LMA-{YYYYMM}-{NNN}.pdf
```

### 代替案 A: Pandoc + wkhtmltopdf

```bash
pandoc diagnosis/cases/LMA-202604-001.md \
  --pdf-engine=wkhtmltopdf \
  --css diagnosis/style.css \
  -o diagnosis/cases/LMA-202604-001.pdf
```

### 代替案 B: ブラウザの「印刷 → PDF で保存」

1. VS Code で `.md` を開き、Markdown Preview Enhanced 等でプレビュー
2. プレビューを右クリック → 「PDF に書き出し」
3. A4、カラー印刷、余白「狭く」で出力

### 代替案 C: オンライン変換

- https://md2pdf.netlify.app/ にペースト → Download PDF
- https://www.markdowntopdf.com/ も可

いずれの方法でも、**PDF を開いて全 5 ページ分の体裁を目視確認** する。

## 📊 7. アフターフォロー（24-72 時間後）

送信から **3 営業日以内** に、「ご返信が無い = 満足してくださっている」
とは限らない。以下の分岐で動く:

### 7.1 返信あり・興味あり
→ サービス案内の個別メールで次サービス（Layer 01-03）へ誘導

### 7.2 返信あり・満足のみ
→ 「また必要な時はいつでも」の 1 行返信。感謝を伝える

### 7.3 返信なし（3 営業日経過）
→ 軽い follow-up を 1 回だけ送る:

```
件名: Re: 【Loomia】AI 活用度診断レポートをお届けしました

{お客様のお名前} 様

先日お送りしたレポートはご覧いただけましたでしょうか。
不明点や「この施策だけもう少し詳しく」といった
ご質問がございましたら、お気軽にご返信ください。

引き続きよろしくお願いいたします。
```

それ以降の追い送信はしない（スパム化を避ける）。
